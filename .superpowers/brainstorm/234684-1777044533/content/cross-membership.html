People in multiple workgroups — how should they appear?

De Woonwolk has many people in 2–3 workgroups. This choice shapes everything else.

A — One node per person

Each person appears exactly once. Multiple workgroup edges pull them toward each group. Multi-members naturally sit between their workgroups, at the intersection. Half-half colour or ring shows the split.

✓ No duplicates  ·  ✓ Cross-members visually stand out at intersections  ·  ✓ Clean for "find a person"  ·  ⚠ Dense centre if many overlaps

B — Clone nodes per membership

Each person gets one node per workgroup. Robin in Coord+Finance = two Robin nodes, linked by a dashed "same person" line. Each cluster stays tight. Clicking any clone highlights all clones.

✓ Each workgroup cluster stays clean  ·  ✓ Clear "who's in this group"  ·  ⚠ Same person appears 2–3×  ·  ⚠ Graph grows larger with 45 members
