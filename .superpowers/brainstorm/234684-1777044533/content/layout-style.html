What graph layout feels right?

Three fundamentally different ways to show a community's structure. Each has different trade-offs for readability, interactivity, and density.

A — Force-directed clusters

Workgroup nodes act as gravity centres. People float around their workgroup(s). Cross-membership shown by dashed links. Physics simulation — it breathes and settles. Feels like a constellation.

✓ Great for discovering who's connected  ·  ✓ Handles multi-workgroup members naturally  ·  ⚠ Can be chaotic with 45+ nodes

B — Bubble / enclosure

Each workgroup is a bubble containing its people. Overlapping bubbles show shared members. Very readable structure — like a Venn diagram of belonging.

✓ Crystal-clear group membership  ·  ✓ Beautiful for print/export  ·  ⚠ Overlaps get messy with many cross-memberships

C — Radial v2 (structured spokes)

Improved version of the current approach. Community at centre, workgroups on spokes, people fanned out near their workgroup. Clean hierarchy, immediately readable. No physics.

✓ Clear hierarchy  ·  ✓ Predictable layout  ·  ⚠ Cross-members hard to show  ·  ⚠ Spokes crowd at >8 workgroups
