# CORE — Coolify Deployment Guide

Deploy the full CORE stack (Node/Express API + React frontend + PostgreSQL) on a VPS using Coolify.

**Architecture:** single Docker container (API + frontend bundled) + one managed PostgreSQL database, both inside Coolify on the same server.

---

## Prerequisites

- Coolify installed and running on your VPS
- Domain pointed at your VPS (e.g. `core.serzhilin.nl` → your server IP)
- GitHub repo with CORE pushed to `main`

---

## Step 1 — Create a Project in Coolify

1. Open Coolify → click **Projects** in the left sidebar
2. Click **+ Add** → name it `CORE`
3. Click into the project

> All resources (database + app) must live in the same project so they can talk on the internal network.

---

## Step 2 — Add PostgreSQL Database

1. Inside the CORE project, click **+ New Resource**
2. Choose **Database** → **PostgreSQL**
3. Fill in:
   - **Name:** `core-db`
   - **Postgres User:** `core`
   - **Postgres Password:** *(generate: `openssl rand -hex 24`)*
   - **Postgres DB:** `core`
4. Leave everything else default → click **Save**
5. Click **Start** and wait until **Running** (green)
6. Click the database resource → **Connection** tab
7. Copy the **Internal Connection URL** — looks like:
   ```
   postgresql://core:<password>@core-db:5432/core
   ```
   > ⚠️ The hostname (`core-db` or similar) is generated by Coolify. Use exactly what Coolify shows.

---

## Step 3 — Add the Application

1. Inside the CORE project, click **+ New Resource**
2. Choose **Application** → **Git**
3. Enter your repository URL (e.g. `https://github.com/Serzhilin/CORE.git`)
4. Set **Branch:** `main`
5. Coolify detects `Dockerfile` → Build Pack shows **Dockerfile**
6. Click **Save**

---

## Step 4 — Configure Port

1. In the application resource, go to **Configuration**
2. Set **Ports Exposes** to: `3002`
3. Click **Save**

---

## Step 5 — Assign Domain

1. Still in **Configuration** → find **Domains**
2. Delete any auto-generated domain
3. Add: `https://core.serzhilin.nl` (or whatever your domain is)
4. Click **Save**

---

## Step 6 — Set Environment Variables

Go to the **Environment Variables** tab. Add each variable:

| Variable | Value | Build | Runtime |
|---|---|---|---|
| `NODE_ENV` | `production` | ❌ | ✅ |
| `PORT` | `3002` | ❌ | ✅ |
| `DB_HOST` | *(hostname from Step 2 internal URL)* | ❌ | ✅ |
| `DB_PORT` | `5432` | ❌ | ✅ |
| `DB_USER` | `core` | ❌ | ✅ |
| `DB_PASSWORD` | *(your password from Step 2)* | ❌ | ✅ |
| `DB_NAME` | `core` | ❌ | ✅ |
| `DB_SYNCHRONIZE` | `true` | ❌ | ✅ |
| `JWT_SECRET` | *(generate: `openssl rand -hex 64`)* | ❌ | ✅ |
| `PUBLIC_REGISTRY_URL` | `https://registry.w3ds.metastate.foundation` | ❌ | ✅ |
| `VITE_PUBLIC_CORE_BASE_URL` | `https://core.serzhilin.nl` | ❌ | ✅ |

> `DB_SYNCHRONIZE=true` creates tables on first boot. Remove it after first successful deploy.

Click **Save** after all variables.

---

## Step 7 — Deploy

1. Click **Deploy** (top right)
2. Watch **Deployment Logs** — build takes 2–3 minutes
3. Successful deploy ends with:
   ```
   CORE API running on :3002
   ```
4. Open `https://core.serzhilin.nl` — you should see the CORE login screen

---

## Step 8 — After First Successful Deploy: Disable DB_SYNCHRONIZE

1. Go to **Environment Variables**
2. Find `DB_SYNCHRONIZE` → uncheck **Runtime Variable**
3. Click **Save** → click **Restart** (not redeploy)

---

## Step 9 — Seed Initial Data (first time only)

1. In Coolify → application resource → **Terminal** tab (or SSH into VPS):
   ```bash
   docker exec -it <container-name> node dist/seed.js
   ```
   Find container name under **Containers** tab.

---

## Troubleshooting

### "Page not found"
→ Domain not assigned. Configuration → Domains → add your domain.

### Build fails at `npm ci`
→ Check build logs for exact error.

### Container crashes on start
→ Check **Logs** tab. Likely causes:
- `JWT_SECRET` missing
- `DB_HOST` wrong — copy exact hostname from Coolify's Connection tab, not `localhost`
- `DB_PASSWORD` mismatch

### eID login doesn't work
→ `VITE_PUBLIC_CORE_BASE_URL` must be your real public domain. The eID wallet needs to reach your server for the callback.

---

## Re-deploying After Code Changes

Push to `main` → click **Redeploy** in Coolify (or enable auto-deploy in settings).
